# Security Checklist — [project-slug] v[X.Y.Z]

> Запускать перед каждым релизом в продакшн / TestFlight / публичным деплоем.

---

## 🔑 Секреты и ключи

- [ ] Нет хардкода API ключей, паролей, токенов в коде
- [ ] `.env` файлы в `.gitignore`
- [ ] `git log --all -p | grep -i "secret\|password\|api_key\|token"` — ничего не найдено
- [ ] Все env-переменные задокументированы в `.ai-codex/env.md`

## 🛡️ Аутентификация и авторизация

- [ ] Все приватные endpoints защищены auth middleware
- [ ] Нет публичных endpoints которые должны быть приватными
- [ ] JWT/session токены имеют срок жизни
- [ ] Нет bypass-маршрутов оставшихся от разработки

## 🧹 Входные данные

- [ ] Все пользовательские inputs валидируются на сервере (не только на клиенте)
- [ ] SQL запросы используют параметризацию (не конкатенацию строк)
- [ ] Загрузка файлов — проверяется тип и размер
- [ ] API принимает только ожидаемые поля (no mass assignment)

## 🌐 Сеть и транспорт

- [ ] HTTPS везде (нет HTTP fallback в продакшне)
- [ ] CORS настроен на конкретные домены (не `*`)
- [ ] Rate limiting на auth endpoints
- [ ] Sensitive данные не логируются (пароли, токены, PII)

## 📦 Зависимости

- [ ] `npm audit` / `pip audit` / `flutter pub outdated` — нет критических уязвимостей
- [ ] Нет неиспользуемых пакетов с известными CVE

## 📱 Mobile (Flutter — если применимо)

- [ ] Нет чувствительных данных в логах (`print`, `debugPrint`)
- [ ] Keychain/Keystore используется для токенов (не SharedPreferences/UserDefaults)
- [ ] Deep links валидируются
- [ ] Certificate pinning (если требуется)

## 🗄️ База данных

- [ ] RLS (Row Level Security) включён для Supabase таблиц
- [ ] Миграции обратимы (есть `down` вариант)
- [ ] Backup перед деплоем с миграцией
