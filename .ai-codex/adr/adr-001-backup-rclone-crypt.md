# ADR-001: Зашифрованный облачный бэкап через rclone crypt

> Файл: `.ai-codex/adr/adr-001-backup-rclone-crypt.md`

---

**Статус:** `proposed`
**Дата:** 2026-06-12
**Проект:** obsidian-second-mind

---

## Контекст

Vault `~/SecondMind` — единственное место накопленной памяти системы (659 markdown, temporal KG, снапшоты сессий). Локального бэкапа нет: если диск Mac полетит — память потеряна навсегда. Нужен облачный бэкап, который:

1. **Занимает мало места** — пользователь явно просил.
2. **Шифрован** — в vault лежит `botseller/tron-wallet-mnemonics--botseller-three-products.md` (мнемоники крипто-кошелька). Plaintext в облаке = недопустимо.
3. Простой в настройке, работает фоном на Mac.

Ключевой инсайт по размеру: ChromaDB-индекс (`~/.obsidian-bridge/chroma`) лежит **вне vault** и **реиндексируется** из текста (`obsidian-bridge index`). Значит бэкапить нужно только текст → 3.4M → gzip ~600KB–1MB.

## Рассмотренные варианты

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **A. rclone + crypt → Google Drive** | Шифрование на лету (AES, ключ только локально); любое облако (Drive/S3/Dropbox); инкрементально; mature; ротация из коробки; ставится `brew install rclone` | Нужна разовая настройка crypt-remote; не установлен сейчас |
| **B. Git private repo (GitHub)** | Версионирование/diff бесплатно; знакомый flow | Личные заметки + **мнемоники** на GitHub-серверах (даже private); история раздувается; шифрование требует git-crypt поверх — лишний слой; коммит-шум |
| **C. Google Drive API напрямую (Python)** | Без внешних бинарей | OAuth-морока; шифрование пишем сами; больше кода для того, что rclone уже решил; привязка к одному облаку |
| **D. Статус-кво (нет бэкапа)** | — | Потеря всей памяти при сбое диска |

## Решение

**Выбрано: Вариант A — rclone + crypt → Google Drive.**

rclone `crypt`-backend шифрует архив локально перед отправкой (мнемоники защищены), даёт ротацию и не привязывает к одному облаку, при минимуме своего кода — мы вызываем `rclone copy`, а не пишем OAuth и шифрование руками.

## Последствия

**Положительные:**
- Мнемоники и личные заметки уезжают в облако **только в зашифрованном виде**; ключ (`rclone.conf`) — локально, chmod 600.
- Архив < 1.5 MB за счёт исключения ChromaDB → дёшево по месту и трафику.
- Облако меняется одной строкой конфига rclone (Drive → S3/Dropbox) без переписывания кода.
- Восстановление детерминировано: download → decrypt → untar → `obsidian-bridge index`.

**Отрицательные / trade-offs:**
- Нет дешёвого пофайлового версионирования как в git (митигируем ротацией N точек: 7 daily + 4 weekly).
- Разовая ручная настройка rclone crypt-remote (документируется в README).
- Внешняя зависимость — бинарь `rclone` (не Python-пакет).

**Что потребует изменений:**
- `brew install rclone` + однократный `rclone config` (создать crypt-remote поверх Google Drive).
- Новый `src/obsidian_bridge/backup.py`, команда `cli.py backup [--restore]`.
- LaunchAgent `dev.nzt108.obsidian-backup.plist` (ежедневно).
- **fail-closed**: backup отказывается стартовать, если crypt-remote не настроен (защита мнемоник).
- README: раздел «Backup & Restore».
